# ADR-002: ProcessingPipeline 单次内存遍历，一次写入

## 状态
已接受（2026-07），修订（2026-07-18）

## 背景
旧版本（v1.3.x 及更早）的处理流程是"每步写盘→下一步再读回"模式。行合并写一次中间文件，CJK 修复再读入→修改→写出，标点替换再读入→修改→写出。这导致：
- 大文件多次 I/O 性能差
- 中间文件散落，最后还得手动清理
- 各步骤耦合在文件系统上，难以单元测试

## 决策
`ProcessingPipeline` 将所有后处理步骤（行合并 → CJK 修复 → 标点截断修复 → 去除行首逗号 → 标点替换）串联为一次性内存管道：
1. `LineMerger.Merge()` 返回 `(List<string> Lines, string OutputPath)` — 内存行列表 + 最终输出路径
2. 后续每个 Merger 都操作 `List<string>` 原地修改
3. 仅在最后一步调用 `File.WriteAllLines()` 一次写入

## 后果
- **优点**：3+2 次文件 I/O → 1 次写入，大文件性能大幅提升
- **优点**：清理简化，没有中间文件残留
- **优点**：所有 Service 类都是纯内存操作，可单元测试
- **缺点**：大文件全部驻留内存（当前场景文本文件通常 < 50MB，可接受）
- **缺点**：如果某个步骤崩溃，无法从中间状态恢复
